* URL의 이해* - URL이 뭔데? 

url을 통해 링크로 이동한다 -> 정적인 파일을 가져온다
 ex) 5개의 페이지 이동 링크 -> 5개의 정적인 파일 가지고 있어야 함


URL은 어떻게 구성돼있어? 
 - 프로토콜(통신규칙)://도메인이름&호스트:포트넘버/실행 중인 파일명?query string
  : protocol 통신규칙 - 사용자가 서버에 접속할 때 쓸 방식(ex: http - hyper text tarnsfer protocol)
  : host(domain) - host는 인터넷에 접속되어 있는 각각의 컴퓨터, 혹은 컴퓨터의 주소
  : port - 한 컴퓨터에 여러개의 서버가 있는 경우, 클라이언트가 접속할 서버가 연결되어 있는 포트 번호이다.
  : path - 현재 디렉토리 경로 안에 있는 실행 중인 파일 명
  : query string - 웹서버에게 '데이터를 전달할 수 있다.'
    (ex: id=HTML&page=120 => 내가 읽고 싶은 정보는 html이고 12페이지이다)
  : query string의 시작은 '?'이고, 값과 값의 구분은 '&', 값의 이름과 값의 구분은 '='로 구분
 - WEB서버는 굉장히 유명한 서버임. 즉, 전세계적으로 쓰는 포트 번호가 동일함.
 - http를 통한 접속 => 포트번호(80) 생략 후 접속 가능(기본값이 80이라는 소리)
